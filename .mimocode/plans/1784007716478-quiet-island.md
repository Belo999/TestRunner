# TestRunner - What Has Been Built So Far

## Project Overview

**MarathonRunner Enterprise** is a Kubernetes-native performance testing platform, scaffolded as an enterprise-grade alternative to LoadRunner Enterprise. The current repository is the **third implementation slice** — a locally runnable scaffold providing the full platform shell with real engine execution via Docker containers.

---

## Architecture

```
                    ┌──────────────────────┐
                    │   Web Browser (SPA)  │
                    │   index.html/app.js  │
                    └──────────┬───────────┘
                               │ HTTP
                    ┌──────────▼───────────┐
                    │   MarathonRunner API  │
                    │   (server.py)        │
                    │   ThreadingHTTPServer │
                    └──┬────┬────┬────┬───┘
                       │    │    │    │
            ┌──────────┘    │    │    └──────────┐
            ▼               ▼    ▼               ▼
      ┌──────────┐   ┌──────────┐  ┌──────────┐  ┌──────────┐
      │ PostgreSQL│   │  Redis   │  │  MinIO   │  │  Docker  │
      │ / SQLite │   │ (cache)  │  │ (S3)     │  │  Engine  │
      │          │   │          │  │          │  │Containers│
      └──────────┘   └──────────┘  └──────────┘  └────┬─────┘
                                                       │
                    ┌──────────────────────────────────┘
                    │  Worker (worker.py)
                    │  - queue_ready_runs
                    │  - launch_engine (Docker)
                    │  - collect_engine_results (parser)
                    └──────────────────────────────────┘
```

Key design decisions:
- **No web framework** — pure Python stdlib `http.server`
- **No ORM** — raw SQL with dual-backend abstraction (SQLite local / PostgreSQL production)
- **Minimal dependencies** — raw TCP for Redis, raw urllib for MinIO, raw subprocess for Docker
- **Two service roles** in one codebase — API server and Worker, switched via `MARATHONRUNNER_SERVICE_ROLE` env var

## What's NOT Built Yet (from Backlog)

Based on the 705-line backlog (`Implementation/32-Full-Project-Implementation-Backlog.md`):

### Missing: Tests
- **Zero automated tests** — no unit tests, no integration tests, no test framework configured
- No `tests/` directory, no `pytest.ini`, no `package.json` for frontend tests

### Missing: Real AI Integration
- AI insights are currently **simulated** (random/hardcoded data in `models.py`), not connected to any LLM
- Anomaly detection, bottleneck correlation, run summaries — all placeholder logic

### Missing: Kubernetes-Native Execution
- Currently runs engines via local Docker, not Kubernetes pods
- No K8s operator, no CRDs, no KEDA autoscaling

### Missing: Enterprise Integrations
- No Jira, ServiceNow, Slack, Teams integrations (webhook model exists but no real connectors)
- No GitOps-based configuration

### Missing: Production-Grade Features
- No OpenTelemetry correlation
- No chaos testing
- No multi-region support
- No cost-aware scheduling
- No data residency controls
- No service virtualization

### In Progress
- Admin panel UI refresh was recently completed (per `TODO.md`)
