# Engine Execution Implementation Plan

## Goal

Add real Docker-based engine execution to MarathonRunner so JMeter and k6 tests actually run and produce real results, replacing the current simulated worker.

## Architecture Decisions

- **Python stdlib only** — no new dependencies (subprocess for Docker, xml.etree for JMeter JTL parsing, json for k6 output)
- **Docker CLI via subprocess** — simpler than Docker SDK, no pip install needed, works everywhere Docker is installed
- **Keep SQLite** — no Postgres migration yet
- **Engine abstraction** — pluggable adapter pattern so Gatling/Locust/Playwright can be added later
- **Volume-based result collection** — mount a host directory into engine containers, parse results from there

## Step 1: Create Engine Abstraction (`apps/api/engines/`)

### `engines/base.py` — Abstract interface

```python
class EngineResult:
    p50_ms: int
    p95_ms: int
    p99_ms: int
    throughput_rps: float
    error_rate: float
    raw_output_path: str | None

class Engine(ABC):
    @abstractmethod
    def image(self) -> str: ...

    @abstractmethod
    def build_run_command(self, run_config: dict) -> list[str]: ...

    @abstractmethod
    def parse_results(self, result_dir: str) -> EngineResult: ...

    @abstractmethod
    def artifact_glob(self) -> str: ...
```

### `engines/registry.py` — Maps engine names to adapters

```python
ENGINES = {"JMeter": JMeterEngine, "k6": K6Engine}
```

### `engines/jmeter.py` — JMeter adapter

- **Image**: `justb4/jmeter:latest`
- **Command**: `jmeter -n -t /test/plan.jmx -l /results/results.jtl -e -o /results/report`
- **Volumes**: mount test script dir + results dir
- **Parse**: read `/results/results.jtl` (XML format), extract timestamps and response times

### `engines/k6.py` — k6 adapter

- **Image**: `grafana/k6:latest`
- **Command**: `k6 run --summary-export=/results/summary.json /scripts/test.js`
- **Volumes**: mount script dir + results dir
- **Parse**: read `/results/summary.json` (k6 built-in summary export)

## Step 2: Create Parsers

### `engines/jmeter_parser.py`

Parse JMeter JTL XML output:
```xml
<testResults>
  <sample ts="..." t="245" ... />
</testResults>
```
Extract: all response times → compute p50/p95/p99, count errors, compute throughput from time span.

### `engines/k6_parser.py`

Parse k6 JSON summary:
```json

    "http_reqs": { "values": { "rate": 45.2 } },
    "http_req_failed": { "values": { "rate": 0.02 } }
  }
}
```

## Step 4: Enhance Worker for Real Execution

### Database change
Add `execution_id TEXT` column to `test_runs` table to store Docker container ID.

### Worker tick flow

```
process_worker_tick():
  1. queue_ready_runs()           # same as before
  2. For each queued run:
     - Start it (status → running)
     - Launch Docker container via engine adapter
     - Store container ID in execution_id column
  3. For each running run:
     - Check if container is still running (docker inspect)
     - If container finished:
       - Collect result files from mounted volume
       - Parse results via engine parser
       - Store metrics in run_results
       - Update run status → completed
       - Release pool capacity
     - If container failed:
       - Update run status → failed
       - Store error info
     - If timeout exceeded:
       - Stop container
       - Update run status → failed
```

### Docker execution details

For each run, the worker:
1. Creates a result directory: `data/artifacts/run-{id}/`
2. Copies the test script into the result directory
3. Runs: `docker run --rm -v {result_dir}:/results -v {script_dir}:/scripts {image} {command}`
4. Stores the container ID from `docker create` or tracks via `docker run -d`
5. On completion, reads result files from `data/artifacts/run-{id}/`

### Timeout enforcement
Use `MARATHONRUNNER_WORKER_COMPLETE_SECONDS` (default 300 = 5 min) for max execution time. If exceeded, `docker kill {container_id}`.

## Step 6: Docker Compose Changes

Add Docker socket mount to `marathonrunner-app`:
```yaml
marathonrunner-app:
  volumes:
    - marathonrunner-data:/app/data
    - /var/run/docker.sock:/var/run/docker.sock  # NEW
```

Add a separate worker service:
```yaml
marathonrunner-worker:
  build:
    context: .
    dockerfile: apps/api/Dockerfile
  environment:
    MARATHONRUNNER_SERVICE_ROLE: worker
    MARATHONRUNNER_DB_PATH: /app/data/marathonrunner.db
    MARATHONRUNNER_WORKER_COMPLETE_SECONDS: "300"
  volumes:
    - marathonrunner-data:/app/data
    - /var/run/docker.sock:/var/run/docker.sock
  depends_on:
    - marathonrunner-app
```
