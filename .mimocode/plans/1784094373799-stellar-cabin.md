# Kubernetes Operator for MarathonRunner Enterprise

## Context

MarathonRunner currently runs test engines (JMeter, k6, Gatling, Locust, Playwright) as Docker containers via `subprocess.run(["docker", "run", "-d", ...])` from a worker process. This works for local/Compose deployments but does not scale to Kubernetes clusters. The Kubernetes Operator will make test execution declarative and cloud-native by introducing Custom Resource Definitions (CRDs) and a reconciliation loop.

## Goal

Build a Kubernetes Operator in Go that:
1. Defines `TestRun` and `PerformanceTest` CRDs
2. Reconciles `TestRun` CRs into Kubernetes Jobs (one Job per engine pod)
3. Monitors Job status and updates the CR status
4. Reports results back to the MarathonRunner API
5. Cleans up completed Jobs

## Architecture

```
User -> MarathonRunner API -> creates TestRun CR via K8s API
                                    |
                          Operator (reconcile loop)
                                    |
                     Creates/manages Kubernetes Jobs
                                    |
                          Engine pods run tests
                                    |
                     Operator watches Job status
                                    |
                     Reports results to MarathonRunner API
```

  namespace: marathonrunner-execution

  durationMinutes: 5

### Phase 2: TestRun Reconciliation Controller

Reconciliation logic for `TestRun`:

```
Reconcile(req):
  1. Fetch TestRun CR
  2. If being deleted -> clean up Job, remove finalizer
  3. Switch on status.phase:
     - "" (new):
       a. Create engine Job from engine adapter's build_k8s_job_spec()
       b. Set status.phase = Pending, status.jobName = job.Name
     - Pending:
       a. Check if Job exists and is running
       b. If running -> set status.phase = Running
       c. If failed -> set status.phase = Failed
     - Running:
       a. Check Job status
       b. If succeeded -> collect results, set status.phase = Succeeded
       c. If failed -> set status.phase = Failed
       d. If exceeded duration -> delete Job, collect partial results
     - Succeeded/Failed:
       a. Ensure finalizer is removed
       b. Optionally clean up Job after retention period
```

### Phase 3: Engine Adapter K8s Integration

Each engine adapter gets a `build_k8s_job_spec()` method:

**Modified files:**
- `apps/api/engines/base.py` - Add abstract `build_k8s_job_spec()` method
- `apps/api/engines/k6.py` - k6 Job spec
- `apps/api/engines/jmeter.py` - JMeter Job spec
- `apps/api/engines/gatling.py` - Gatling Job spec
- `apps/api/engines/locust.py` - Locust Job spec
- `apps/api/engines/playwright.py` - Playwright Job spec

Example for k6:
```python
def build_k8s_job_spec(self, run_config: dict) -> dict:
    return {
        "apiVersion": "batch/v1",
        "kind": "Job",
        "metadata": {
            "name": f"mr-run-{run_config['run_id']}-k6",
            "namespace": run_config.get("namespace", "marathonrunner-execution"),
            "labels": {
                "marathonrunner.io/run-id": str(run_config["run_id"]),
                "marathonrunner.io/engine": "k6",
            },
        },
        "spec": {
            "backoffLimit": 0,
            "ttlSecondsAfterFinished": 3600,
            "template": {
                "spec": {
                    "restartPolicy": "Never",
                    "containers": [{
                        "name": "k6",
                        "image": "grafana/k6:latest",
                        "command": ["k6", "run",
                                    "--summary-export=/results/summary.json",
                                    "/scripts/test.js"],
                        "env": [
                            {"name": "VUS",
                             "value": str(run_config["target_vusers"])},
                            {"name": "DURATION",
                             "value": f"{run_config['duration_minutes'] * 60}s"},
                            {"name": "TARGET_ENDPOINT",
                             "value": run_config["target_endpoint"]},
                        ],
                        "volumeMounts": [
                            {"name": "scripts", "mountPath": "/scripts",
                             "readOnly": True},
                            {"name": "results", "mountPath": "/results"},
                        ],
                        "resources": run_config.get("resources", {}),
                    }],
                    "volumes": [
                        {"name": "scripts",
                         "configMap": {"name": run_config["script_configmap"]}},
                        {"name": "results", "emptyDir": {}},
                    ],
                },
            },
        },
    }
```

### Phase 4: API Integration

**Modified files:**
- `apps/api/server.py` - Add K8s mode endpoints
- `apps/api/worker.py` - Add K8s execution path
- `apps/api/models.py` - Add K8s status sync functions

New API endpoints:
- `POST /api/runs/{id}/k8s-launch` - Create TestRun CR
- `GET /api/runs/{id}/k8s-status` - Get TestRun CR status
- `GET /api/k8s/nodes` - List K8s cluster nodes
- `GET /api/k8s/jobs` - List marathonrunner Jobs

Worker changes:
- Add `MARATHONRUNNER_EXECUTION_MODE` env var (`docker` or `kubernetes`)
- In K8s mode, worker creates TestRun CR instead of Docker container
- Worker polls K8s API for Job status instead of `docker inspect`
