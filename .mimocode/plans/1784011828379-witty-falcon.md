# Plan: REST API Endpoint Tests for server.py

## Goal

Write comprehensive tests for `apps/api/server.py` (0% coverage, 589 lines, 78 endpoints) to bring it to ~80%+ coverage. This is the single biggest coverage gap in the project.

## Approach

Start a real `ThreadingHTTPServer` per test on a random port, use `urllib.request` to make HTTP requests. This tests the full request/response cycle through the actual handler routing.

## File to Create

`tests/test_server.py` (~150 tests across 31 classes)

def api_get(url, token=None):
    req = urllib.request.Request(url)
    if token:
        req.add_header("Authorization", token)
    try:
        resp = urllib.request.urlopen(req)
        return resp.status, json.loads(resp.read())

## Key Implementation Details

1. **DB_PATH monkeypatch**: Must set BOTH `monkeypatch.setattr("apps.api.database.DB_PATH", db_path)` AND `monkeypatch.setenv("MARATHONRUNNER_DB_PATH", str(db_path))` — the env var is read at import time, the attr at call time.

2. **SERVICE_ROLE**: Module-level `os.environ.get()` in server.py — must monkeypatch `apps.api.server.SERVICE_ROLE` for worker tick tests.

3. **Seed data**: Must insert runs with `status='completed'` and matching `run_results` rows for baseline/compare/report tests. Must insert a `status='pending_approval'` run for the approve endpoint.

4. **Per-test server**: `ThreadingHTTPServer(("127.0.0.1", 0))` starts on random port. ~50ms overhead per test, gives perfect isolation.

5. **Auth bypass**: Public paths (/api/health, /api/auth/login, /docs, /swagger, /api/openapi.json) must work without tokens.

6. **Error paths**: Each endpoint group should test at least: missing auth (401), wrong role (403), invalid body (400), not found (404).
